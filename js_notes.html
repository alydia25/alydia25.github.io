Notes for Webdev JS Section

Chapter 1: Bird's Eye View

JavaScript (JS) is considered web coding and primarly client side

Pretty much universal with few exceptions

Only need a text editor and a browser to code JS, programs to help edit are reccomended but not required

Use // for single line comments in JS code

Use /* and */ for multi line comments in JS code

Can use script tag in header to link exernal JS, similar to CSS, or infile between the script tags.

Short infile lines are okay, but if the JS code is long, then put it in a seprate file to avoid clutter

If using external, must be plain text format, use the .js extension, DO NOT use script tag in the file, jsut statements as one would within the HTML

Statements outside of functions executed automatically when ref is discovered, statements within a function not executed until function is called.

Chapter 2: Understanding Variables

Can declare a variable with let

Can only declare a variable once, although you can use it mutiple times

Use comments to describe variables

Declare variables before using!

Declare the variable as close as possible to where it is first used

Store a variable with setting variableName = value;

const variableName = value; is another way to set values

Const vs Let:

Using Const is a nonchanging value and helps with remembering, trying to assign a new one will get an error with this type

Only use let for variables that will change

When getting data from user, use prompt(type, initialValue); with value type as type, and initialValue as optional for what appears in the prompt box

prompt(); will always return a value, if cancelled, will return null

For naming, first character must be an underscore or a letter, but never a number

The rest of the variable name can be whatever you want, but no spaces, symbols, punctuiation marks, or other characters

Variable names are case sensitive

No length limit to name

Cannot use a "reserved word" as a name (ex: let, const, var, alert, or prompt)

Make names descriptive

Avoid single letter names

Use mutiple words with no spaces

Use different naming conventions for HTML, CSS, and JS

Differentiate variable naems from JS keywords

Do not make names too long

Data Types are very important

intergers and floating-point numbers are numeric

floating-point can use exponential notation

Hexadecimal intergers (base-16)

String Literals is a sequence of one or more numbers, letters, or punctuation parks in either double or single quotes

Illegal to insert in a strong one or more instances of same quotation mark that encloses the string

Escape Sequences
Escape Sequence Character it Represents
\' single quotation mark
\" double quotation mark
\b backspace
\f form feed
\n new line
\r carrige return
\t tab
\\ backslash

Boolean literals are tied directly to a variable, either true or false

Chapter 3: Building Expressions

Expressions are a collection of symbols, words, and numbers that performs a calculation and produces a result.

Operands are input values for an expression

Operators are symbols that represent a particular action performed on one (or more) operands

x = x + y can also be x += y

Incrementing allows to add one to a variable by using ++

Decrement does the same thing but minus one.

You can also break up long statements with "+"
